  📋 LANGKAH-LANGKAH (PILIH SALAH SATU):
═══════════════════════════════════════════════════════════════════

OPSI 1: HARD REFRESH (TERCEPAT) ⚡
  → Tekan: Ctrl + Shift + R (Windows/Linux)
  → Atau: Cmd + Shift + R (Mac)

OPSI 2: CLEAR CACHE MANUAL 🧹
  1. Tekan Ctrl + Shift + Delete
  2. Pilih "Cached images and files"
  3. Time range: "All time"
  4. Klik "Clear data"
  5. Refresh dengan F5

OPSI 3: INCOGNITO MODE (PALING AMAN) 🕵️
  1. Tekan Ctrl + Shift + N
  2. Buka http://localhost:5173
  3. Test di incognito window

═══════════════════════════════════════════════════════════════════
  ✅ CARA CEK BERHASIL:
═══════════════════════════════════════════════════════════════════

1. Buka http://localhost:5173
2. Tekan F12 untuk buka Console
3. Harus ada log ini:

   [Auth] Development mode - auto-login as dev user
   [Realtime] Development mode - realtime subscription DISABLED
   [AppContext] Loading keywords from Supabase...
   [API] GET /keywords
   [AppContext] Keywords loaded: X items

4. Ubah status tender (Dipantau → Diikuti)
5. Refresh halaman (F5)
6. Status harus tetap "Diikuti" ✅

═══════════════════════════════════════════════════════════════════
  ❌ JIKA MASIH ERROR:
═══════════════════════════════════════════════════════════════════

ERROR: "timeout of 3000ms exceeded"
  → Browser masih pakai kode lama
  → Clear cache lagi dengan Opsi 2 atau 3

ERROR: "Anda harus login untuk menyimpan perubahan"
  → Dev mode auto-login belum aktif
  → Clear cache dan hard refresh

ERROR: Data masih revert setelah refresh
  → Cek console untuk error
  → Screenshot dan kirim ke developer
